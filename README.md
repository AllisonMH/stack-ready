# Interview Prep RAG Agent

A retrieval-augmented generation (RAG) Q&A assistant built over a full-stack developer interview study guide. Ask technical interview questions and get answers grounded in the source material, with full visibility into which chunks were retrieved and how relevant they were.

## Stack

- **FastAPI** — HTTP API
- **LangChain + GPT-4o-mini** — LLM orchestration
- **ChromaDB** — local vector store
- **OpenAI text-embedding-3-small** — document embeddings

## Setup

```bash
# 1. Create and activate virtual environment
python -m venv .venv
source .venv/bin/activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Add your OpenAI API key
echo "OPENAI_API_KEY=sk-..." > .env

# 4. Ingest the study guide (one-time)
python ingest.py

# 5. Start the server
uvicorn api:app --reload
```

The API will be live at `http://localhost:8000`.

## Usage

### Ask a question

```bash
curl -X POST http://localhost:8000/ask \
  -H "Content-Type: application/json" \
  -d '{"question": "What is Big O notation?", "k": 4}'
```

`k` controls how many source chunks are retrieved (1–10, default 4).

**Response:**
```json
{
  "answer": "Big O notation is...",
  "sources": [
    {
      "content": "...",
      "source": "study-guide.txt",
      "chunk_index": 42,
      "relevance_score": 0.8712
    }
  ],
  "trace_id": "f3a1...",
  "latency_ms": 1243.5
}
```

### Interactive UI

Open `http://localhost:8000/docs` for the Swagger UI — try questions directly in the browser without curl.

## Observability

Every query is recorded as a trace containing the question, retrieved chunks, relevance scores, answer, and latency.

```bash
# List recent traces (newest first)
GET /traces?limit=20

# Inspect a single trace
GET /traces/{trace_id}
```

Traces are stored in memory (last 100 queries) and reset on server restart.

## Project Structure

```
study-guide.txt   # Source document (full-stack interview study guide)
ingest.py         # Chunk, embed, and load study-guide.txt into ChromaDB
agent.py          # RAG logic: vector search + LLM chain
api.py            # FastAPI endpoints: /ask, /traces, /health
models.py         # Pydantic schemas
observability.py  # Trace store, logger, Timer
chroma_db/        # Persisted vector store (generated by ingest.py, gitignored)
```
