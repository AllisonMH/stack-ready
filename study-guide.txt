Interpretation of Javascript 
* A lightweight programming language that allows you to enhance web pages with interactive and complex features


Javascript prototype & inheritance 
* All javascript objects inherit properties and methods from a prototype 
   * For Example: date objects inherit from Date.prototype
* Object.prototype is at the top
   * All objects inherit from object.prototype 
* The actual prototype property allows you to add new properties to object constructors 
   *      * Prototype property also allows for methods to be added in the same way
   *      * Don’t modify the prototypes of standard JS objects 
   * More info on how this works in ES5+ here: https://www.w3schools.com/js/js_object_es5.asp


Javascript as a prototype based language 
      * Javascript is based on prototypes rather than classes
      * Mainly impacts the hierarchy 
      * Inheritance is performed by reusing existing objects that serve as prototypes instead of “classes” 
      * Use generalized objects which can be cloned and extended 
      * Any object can be associated to the prototype for another object, allowing the second object to share the first object’s properties 
      * Objects are created with a new operator (same as class based)
      * The constructor function or prototype specifies an initial set of properties
      * Properties can be added or removed dynamically to individual objects or an entire set of objects 


JavaScript classes must have a constructor method and can have other functions 
      * Classes must be written with the ‘use strict’ direct
      * To create a class inheritance, use the ‘extends’ keyword 
      * This class inherits all methods from the original class 
      *            * ‘super()’ method refers to parent class 
         * When called in the constructor of the child, you also call the parent’s constructor method and get access to the parent’s properties and methods 
         * Inheritance is important for code reusability 
         * Classes are not hoisted, so they must be declared before use 
         * Two ways of demonstrating classes:
         * Class expressions
  

         * Class declarations

Objects store related values known as properties 
         * Objects are stored by reference

Properties are stored in key value pairs 
            * There are 4 types of properties: 
            * Fields - store values  
            * Methods - perform actions or return the values changed by those actions 
            * Getters - return data 
            * Setters - set/store data 
            * Properties can be accessed using dot notation or bracket notation 
  



            * Properties can also be accessed with destructuring or spread syntax 
  





            * Properties can be enumerated with the Object.keys, object.values, or object.entries methods or with the “for…in” iterator
  



Getters and Setters In Depth
            *                  * Here when you make an assignment, it calls the set method, 
               * When you update the Obj.val, it calls the get to get the current value 
               * Getters and setters are very useful for storing and updating properties
               * In particular if you want to update values before storing them 
               * To add them in a class, must use the ‘get’ and ‘set’ keywords 
               * The name of the getter/setter method cannot be the same as the name of the property 
               * Typically _ is used before property name to indicate the difference 
               * Can call delete to delete a property, but you shouldn’t 
               * Just make a new object without that property


Difference between objects and primitives: 
               * Primitives assign by value, and they copy memory 
               * Objects assign by reference and they share memory 


Useful Pattern for copying objects
               * Shallow copies can be made with Object.assign
               * Object.assign ensures memory is no longer shared and starts at the right end of an object’s list and moves to the left 
               * Spread syntax also does this 
               * There is no discernable difference between spread syntax and obj.assign 
               * For deep copying, it’s best to use JSON deserialization, with JSON.parse(JSON.stringify(object))
               * Works best for simple objects 
               * Lodash can be used for this 
               * Can also write a recursive function to do this

More on JS For..In

Closures in JavaScript
                        * A function having access to the parent scope event after the parent function has closed 
                        *                              * In this example, the variable add is being assigned to a self invoking function.
                           * The counter variable is accessible to and changeable in the function  nested  under it 
                           * The self invoking function only runs once where it sets the counter to 0 and returns the function expression 
                           * Add is function returning another function call 
                           * The counter is protected by the scope of the anonymous function and can only be changed using the add function 




In a Javascript for or while, always set the counting variable external to the actual loop to save space and time

                           * You can’t call a static method on an object, only on an object class

Javascript Variable Hoisting 
                              * A javascript variable can be used before it is declared 
                              * Javascripts default behavior of moving all declarations to the top of the current scope (function or entire script)
                              * Variables declared with let or const are not hoisted 
                              * JS only hoists declarations not initializations 


Global Scope
                              * The area outside of all functions 
                              * Variables declared here can be accessed and altered in any other scopes 


Local Scope
                              * Variables declared inside a function and can only be used in that function 
                              * Local scope can be divided into function scope and block scope 
                              * Function scope - variable is visible only within the function 
                              * Var is used to declare these variables 
                              * Variables declared with var don’t have block scope 
                              * Block scope -  area within if, switches, for, or whiles
                              * Const and let allow you to declare variables here 
                              * Let and const are used and only available in block scope 
                              * KEY DIFFERENCE BETWEEN LET AND CONST -  const once assigned can’t be reassigned to a new value 
                              *   

Lexical Scope
                                 * Children scope have access to variables in the parent scope 
                                 * Children are bound lexically to the execution of their parents (think closures)
  





Event Listeners in Javascript


                                 * addEventListener() attaches an event handler to the specified element
                                 * Parameters are the actual event, and the function you want to be called with that event, the third parameter is a boolean value specifying if the event is to use bubbling or capturing 
                                 * Default value is false, which means to use bubbling 
                                 * When passing parameter values use an anonymous function that calls the specified function with the parameters you’re passing 


Event propagation is a way of defining the element order when an event occurs
                                 * Bubbling - the innermost element’s event is handled first and then the outer 
                                 * Default value in the event listener is false, which means use bubbling 
                                 * Capturing -  the outermost element’s event is handled first and then the inner 
                                 * When the value in the event listener is true, use capturing
                                 * stopPropagation -  prevents propagation of the same event from being called 
                                 * Stops events from bubbling up to parent elements or capturing down to child elements 


Remove Event Listener
                                 * Removes event handlers that have been attached to the addEventListener()
                                 * Syntax - ‘element.removeEventListener(event, function, capture)’


Event Emitter

                                 * ‘Emit’ used to trigger events 
                                 * ‘Once’ one time event listener 
                                 * ‘removeListener’ remove an event listener from an event 
                                 * ‘removeAllListeners’ remove all listeners for an events 
                                 * ‘addListener’ to add an eventListener


ForEach / Filter / Map/ Reduce / .every()  Array Operations 
                                 * All are methods on arrays that are part of the javascript array object 
                                 * Foreach - executes a callback function on each element of an array one by one 
                                 *                                    * Foreach is not promise aware, it will execute all the synchronous stuff and not wait to execute the asynchronous stuff 
                                    * Can’t use await in a foreach 


                                    * Filter - executes a callback for every element and returns a filtered array
                                    * Foreach loops over the array and executes the callback, but filter executes the callback and checks its return value 
                                    * If true, it’s added to the final array 
  



                                    * Doesn’t work with async await because callbacks always return a promise and promises are inherently truthy, therefore every item in the array passes the filter automatically 
                                    * To use actually: use map to return an array of promises, await the array of promises, and filter the resolved values according to the callback 


                                    * Map - executes a callback against every element in an array, but saves the results to a new array
                                    *                                       * If using an await in map, map will return an array of promises and you have to wait for the array of promises to get resolved with an await Promise.all(arrayOfPromises)


                                       * Reduce - reduces an array to a single value 
                                       * For example, adding all the elements in an array 
                                       * Takes in a callback function, the last returned value from the reduce, and performs the operation on that 
                                       * Can work with async/await but you must await the accumulator first or use map to return an array of promises, await the array of promises, and reduce the resolved values 


                                       * More on async/await in loops: https://zellwk.com/blog/async-await-in-loops/


                                       * Array.prototype.every() - checks whether all elements in an array pass the test implemented by the previous function

Sets 
                                          * Collection of values 
                                          * You can iterate through in insertion order 
                                          * A value may only occur once, so all values must be unique 
                                          * Has properties constructor and size 
                                          * Can be created by passing an array of unique values to the method ‘new Set()’
                                          * Can also create an empty set and use the ‘add()’ method to add values 
                                          * If you add equal elements, only the first will be saved 
                                          * Elements of set can be listed with a ‘for..of’ loop 
                                          * Sets are objects and typeof returns object
                                          * Methods: add, clear, delete, entries, forEach, has, keys, values 
                                          * values() returns an iterator object with values in a set
                                          * keys() returns an iterator object with values in a set
                                          * entries() returns an iterator object with [value, value] pairs from a set 
                                          * A set has no keys, so instead of a [key,value] pair, it’s [value, value]
                                          *   

Sets have logic as of JavaScript 2025 
                                             * Methods are 
                                             * union() - returns new set containing union of two sets 
                                             *                                                *                                                   * difference() -  returns a new set containing elements which are in this set, but not the argument set 
                                                *                                                   *                                                      * intersection() - returns a new set containing the elements that are in this set and the argument set
                                                   *                                                      *                                                         * isDisjointFrom() - returns true if this set has no elements in common with the argument set
                                                      *                                                         *                                                            * isSubsetOf() - returns true if all elements in this set are also elements in the argument set 
                                                         *                                                            *                                                               * isSupersetOf() - returns true if all elements in argument set are also in this set 
                                                            *                                                               *                                                                  * symmetricDifference() - returns a new set containing elements which are in this set, or in the argument set, but not in both 
                                                               *                                                                  *   

JavaScript WeakSet is a collection of values where the values must be objects 
                                                                  * It’s called a WeakSet because it hold weak references to its values 
                                                                  * WeakSet does not prevent its values from garbage collected and removed
                                                                  * Primitive values can be values in a WeakSet 
                                                                  * The values must be objects 
                                                                  * Primitives are not garbage collected like objects 
                                                                  * WeakSets are good for storing data like counts 
                                                                  * WeakMaps - collection of key/value pairs where the keys must be objects or non-registered symbols 
                                                                  * Holds weak references to its keys 
                                                                  * Doesn’t prevent keys from being garbage collected 
                                                                  * If a key has no references to it in program, it becomes eligible for garbage collection
                                                                  * Good for tracking data about objects without preventing garbage collection 
                                                                  * Not enumerable 
                                                                  * Can’t iterate over the keys and  values with for loops, forEach(), or keys()
                                                                  * Also can’t access the size 
                                                                  * Has a number of limited methods 
                                                                  * New WeakMap() - creates new weak map object
                                                                  * get(key) -  gets the value for a key in a weak map
                                                                  * set(key, value) - sets the value for a key in a weak map 
                                                                  * delete(key, value) - removes an element specified by a key
                                                                  * has(key) - returns true if a key exists in a weak map 
                                                                  * Doesn’t allow iteration 
                                                                  * Outside cannot discover what objects are stored in a weak map, you need to use the ‘this’ reference

GraphQL is a query language for your API and server-side runtime for executing queries using a type system you define for your data. 


                                                                                                                                                                                                                                                                        * Specification was open sourced in 2015 
                                                                                                                                                                                                                                                                        * GraphQL is not tied to any specific database or storage engine. It’s backed by your existing code and data


A GraphQL service is created by defining types and their fields, then writing a function for each field to provide the required data 


                                                                                                                                                                                                                                                                        * A service that tells you the name of a logged in user may resemble this 
                                                                                                                                                                                                                                                                        *                                                                                                                                                                                                                                                                              * These would be the functions for each field on each type 
                                                                                                                                                                                                                                                                           *                                                                                                                                                                                                                                                                              * The function that provides data for the ‘me’ field on the Query type uses information about the authenticated user who made the request while the name field on the User type is populated by using that user’s ID to fetch their full name from a database
                                                                                                                                                                                                                                                                              * After a GraphQL service is running, usually accessed by a URL on a webservice, it can receive GraphQL queries to validate and execute from clients
                                                                                                                                                                                                                                                                              * The service first checks a query to ensure it only refers to the types and fields defined for the API and then runs the provided functions to produce a result
                                                                                                                                                                                                                                                                              * This query 
                                                                                                                                                                                                                                                                              *                                                                                                                                                                                                                                                                                    * Would produce this result in JSON 
                                                                                                                                                                                                                                                                                 *                                                                                                                                                                                                                                                                                       * The client can make queries to the API that mirror the structure of the data they need and receive just that data in the expected shape with a single request
                                                                                                                                                                                                                                                                                    * GraphQL also allows your API to evolve without managing different API versions 
                                                                                                                                                                                                                                                                                    * No longer supports the deprecated name field 


The GraphQL type system describes what data can be queried from the API 


The collection of those capabilities is referred to as the service’s schema 


 Clients can use that schema to send queries to the API that return predictable results 


GraphQLType Definitions 


GraphQL has six kinds of named type definitions and other features 


GraphQL services can be written in any language and there are many different approaches to take when defining the types in a schema


                                                                                                                                                                                                                                                                                    * Some libraries will have the developer construct the schema types, fields, and resolver functions together using the same programming language that was used to write the GraphQL implementation 


                                                                                                                                                                                                                                                                                    * Some libraries allow the developer to define types and fields using the schema definition language and then write the resolver functions for the corresponding fields separately 


                                                                                                                                                                                                                                                                                    * Some libraries all the developer to write and annotate the resolver function, and then infer the schema 


                                                                                                                                                                                                                                                                                    * Some libraries may even infer both the types and resolver functions on behalf of the developer based on underlying data sources 


The most basic components of a schema are object types which represent the kind of object you can fetch from your service and what fields it has

Algorithms: Binary Trees
                                                                                                                                                                                                                                                                                             * Every node has two children at most 
                                                                                                                                                                                                                                                                                             * Number of nodes in a perfect binary tree is always odd because the first level always has just one node 
                                                                                                                                                                                                                                                                                             * Height is approximately the log base 2 of n+1 over 2 
                                                                                                                                                                                                                                                                                             * Log n is equal to log base 2 of n 


Types of Binary Trees
                                                                                                                                                                                                                                                                                             * Full Binary Tree -  Every node has 0 or 2 children
                                                                                                                                                                                                                                                                                             *   



                                                                                                                                                                                                                                                                                                * Complete Binary Tree - all levels are completely filled except possibly the last level. Last level has all nodes as left as possible 
  





                                                                                                                                                                                                                                                                                                * Perfect Binary Tree - all interior nodes have two children and all leaves are at the same level 
  



                                                                                                                                                                                                                                                                                                * Have to keep binary trees balanced to maintain performance 
                                                                                                                                                                                                                                                                                                * Balanced - maximum height of any branch is no more than 1 more than the minimum height of any branch 
                                                                                                                                                                                                                                                                                                * Tree in which the left and right subtrees of every node differ in height by no more than one 
                                                                                                                                                                                                                                                                                                * Degenerate binary tree -  has same performance level as a linked list 


                                                                                                                                                                                                                                                                                                * Properties of Binary Trees
                                                                                                                                                                                                                                                                                                * Number of leaf nodes = number of nodes with two children + 1 
                                                                                                                                                                                                                                                                                                * Maximum number of nodes at level L = 2 ^ (L-1)
                                                                                                                                                                                                                                                                                                * Maximum number of nodes in a tree of height H = 2 ^(H+1) -1 


Binary Trees are primarily used to implement Binary Search Trees and Binary Heaps 


Algorithms: All Nodes K Distance in Binary Tree 
Given a binary tree with root node root, a target node, and an integer value K
                                                                                                                                                                                                                                                                                                * Return a list of the values of all nodes that are distance K from the target node 
  

Notes:
                                                                                                                                                                                                                                                                                                * Rooted trees are defined by some as a directed graph.
                                                                                                                                                                                                                                                                                                * In graph theory a tree is an undirected graph
                                                                                                                                                                                                                                                                                                * Trees can be traversed by levels 
                                                                                                                                                                                                                                                                                                * A tree is an acyclic, connected graph
                                                                                                                                                                                                                                                                                                * BFS can be done with a queue 


Solution 1 - If we know the parent of every node x, we know that all nodes are distance 1 from x. You can then perform a breadth first search from the target node to find the answer 
                                                                                                                                                                                                                                                                                                * First do a depth first search where you annotate every node with information about its parent
                                                                                                                                                                                                                                                                                                * Afterwards, do a breadth first search to find all nodes distance K from the target 


Solution 2 - Breadth first search
                                                                                                                                                                                                                                                                                                * Caveat, you may have  to travel back to an ancestor node 


                                                                                                                                                                                                                                                                                                * How do you create a mapping of every node’s parent in a directed graph and have constant time access to that mapping for every node?
                                                                                                                                                                                                                                                                                                * Hashtable: do a tree traversal, turn it into an undirected graph, and create a hashtable, then do the breadth first search 


                                                                                                                                                                                                                                                                                                * Using a queue for BFS, a “seen” hashtable, and a current level variable :
                                                                                                                                                                                                                                                                                                * Will make it breadth first search 
                                                                                                                                                                                                                                                                                                * Seen hashtable allows you to differentiate what has already been worked on 
                                                                                                                                                                                                                                                                                                * When the current level is equal to k, then all the nodes in our queue are on the level we want 
                                                                                                                                                                                                                                                                                                * First, turning it into an undirected graph:
                                                                                                                                                                                                                                                                                                * Add the first node to the seen hash table, and the queue, 
                                                                                                                                                                                                                                                                                                * We don’t want the level we are at 
                                                                                                                                                                                                                                                                                                * What is the size of this layer? Not the level that we need 
                                                                                                                                                                                                                                                                                                * We pull the child nodes and parent node of the node we’ve started with 
                                                                                                                                                                                                                                                                                                * Dequeue our start node 
                                                                                                                                                                                                                                                                                                * Add these values (parent and children nodes) to the queue and the seen hash table 
                                                                                                                                                                                                                                                                                                * We’ve explored all the nodes at this layer so now we can increment the current layer level by 1 
                                                                                                                                                                                                                                                                                                * And move on to the next level 
                                                                                                                                                                                                                                                                                                * So we start with one of the child nodes and pull it’s child nodes and parent node  and add those to the queue 
                                                                                                                                                                                                                                                                                                * Parent node is already in the seen hash table 
                                                                                                                                                                                                                                                                                                * Move on to the next child node 
                                                                                                                                                                                                                                                                                                * Add its children to the seen hash table and queue 
                                                                                                                                                                                                                                                                                                * Parent’s already been seen 
                                                                                                                                                                                                                                                                                                * Go back up the parent node node
                                                                                                                                                                                                                                                                                                * It’s left child, the one we started with  is already in the seen table
                                                                                                                                                                                                                                                                                                * So do the right child, not yet in seen and also needs to be added to the queue 
                                                                                                                                                                                                                                                                                                * Next, increment the current level value to 2 
                                                                                                                                                                                                                                                                                                * All values now in queue belong at level 2, therefore these are all the values k distance from our target 
                                                                                                                                                                                                                                                                                                * Bfs time complexity is always number of edges + number of nodes 
                                                                                                                                                                                                                                                                                                * Number of edges will always be one less than the number of nodes 
                                                                                                                                                                                                                                                                                                * Time complexity becomes o(n)
                                                                                                                                                                                                                                                                                                * Space complexity is o(n)

Djikstra’s Shortest Path Algorithm
                                                                                                                                                                                                                                                                                                         * Graphs represent connections between pairs of elements called nodes 
                                                                                                                                                                                                                                                                                                         * The connections themselves are called edges 
                                                                                                                                                                                                                                                                                                         * Two types
                                                                                                                                                                                                                                                                                                         * Directed - you can only go one way from one node to another 
                                                                                                                                                                                                                                                                                                         * Undirected - you can go from one node to another in both directions 
                                                                                                                                                                                                                                                                                                         * Weighted Graphs
                                                                                                                                                                                                                                                                                                         * Edges have a cost or a weight 
                                                                                                                                                                                                                                                                                                         * Weight can represent distance, time, or any model of the connection between two nodes 
                                                                                                                                                                                                                                                                                                         * Weights are essential to Djikstra’s 
                                                                                                                                                                                                                                                                                                         * Djikstra’s is used to find the shortest path from a source node to all other nodes in the path 
                                                                                                                                                                                                                                                                                                         * Starts a node you choose: source node 
                                                                                                                                                                                                                                                                                                         * Keeps track of the currently known shortest distances from each node to the source node 
                                                                                                                                                                                                                                                                                                         * Updates the values if shorter path found 
                                                                                                                                                                                                                                                                                                         * Once algorithm has found the shortest path between the source node and another node, it’s marked as visited and added to the path 
                                                                                                                                                                                                                                                                                                         * Process continues until all nodes have been added to the path
                                                                                                                                                                                                                                                                                                         * This way, there’s a path that connects the source node to all other nodes following the shortest path possible to reach each node 
                                                                                                                                                                                                                                                                                                         * Algorithm only works if graphs have positive weights
                                                                                                                                                                                                                                                                                                         * Negative weights don’t work 


Executing Steps of Dijkstra's Shortest Path Algorithm 


                                                                                                                                                                                                                                                                                                         * The distance from the source node to itself is 0 but distance to all other nodes is unknown
                                                                                                                                                                                                                                                                                                         *    
                                                                                                                                                                                                                                                                                                            * Also have a list to keep track of nodes not yet visited   
                                                                                                                                                                                                                                                                                                               * Since we start at 0, this node has already been visited   
                                                                                                                                                                                                                                                                                                               * Now check for the distance from node 0 to its adjacent nodes   
                                                                                                                                                                                                                                                                                                                  * Update the distances from node 0 to node 1 and node 2 with the weights of the edges that connect them to node 0
                                                                                                                                                                                                                                                                                                                  *                                                                                                                                                                                                                                                                                                                        * After updating the distances of the adjacent nodes we 
                                                                                                                                                                                                                                                                                                                     * Select the node closest to the source node based on known distances, mark it as visited, add it to the path 
                                                                                                                                                                                                                                                                                                                     * In this case 1 has the shortest path so it’s added   
                                                                                                                                                                                                                                                                                                                     * We mark it extra in the list to note that it has been visited and the shortest path found
   
                                                                                                                                                                                                                                                                                                                        * Also mark it off in list of unvisited nodes   
                                                                                                                                                                                                                                                                                                                           * Nodes 3 and 2 are both adjacent to 0 and 1 so they will be next 
                                                                                                                                                                                                                                                                                                                           * 2 and its distance are already in the list so no need to updated them so we just focus on 3
                                                                                                                                                                                                                                                                                                                           *                                                                                                                                                                                                                                                                                                                                 * The distance from node 0 to node 3 is 7
                                                                                                                                                                                                                                                                                                                              * Add the weights of the edges that form the path 0->1->3 = 7
                                                                                                                                                                                                                                                                                                                              * Now we have to choose which node will be added to the path which is the unvisited node with the shortest currently known path to the source node 0 
                                                                                                                                                                                                                                                                                                                              * 2 would be added next because it’s weight is 6   
                                                                                                                                                                                                                                                                                                                                 * Also mark it as visited
                                                                                                                                                                                                                                                                                                                                 *                                                                                                                                                                                                                                                                                                                                       * And remove from unvisited list
                                                                                                                                                                                                                                                                                                                                    *    
                                                                                                                                                                                                                                                                                                                                       * Now repeat to find the shortest path to node 3 
                                                                                                                                                                                                                                                                                                                                       * 3’s distance has been recorded as 7 from node 0 in our list 
                                                                                                                                                                                                                                                                                                                                       * An alternative path could be taken 0->2->3
                                                                                                                                                                                                                                                                                                                                       * Has a total distance of 14   
                                                                                                                                                                                                                                                                                                                                          * The first distance is shorter so we keep the original path 
                                                                                                                                                                                                                                                                                                                                          * Only update if the path is shorter
                                                                                                                                                                                                                                                                                                                                          *    
                                                                                                                                                                                                                                                                                                                                             * Mark node 3 as visited and cross off the unvisited list
                                                                                                                                                                                                                                                                                                                                             *                                                                                                                                                                                                                                                                                                                                                   * Now repeat the process on nodes 4 and 5 
                                                                                                                                                                                                                                                                                                                                                * For node 4, the distance is 17 from path 0->1->3->4 
                                                                                                                                                                                                                                                                                                                                                * For node 5: the distance is 22 from the path 0->1->3->5 
                                                                                                                                                                                                                                                                                                                                                * So we clearly go with the first path 
                                                                                                                                                                                                                                                                                                                                                * We’re extending the shortest path 
                                                                                                                                                                                                                                                                                                                                                * Node 4 will be marked as visited 
                                                                                                                                                                                                                                                                                                                                                *                                                                                                                                                                                                                                                                                                                                                   *                                                                                                                                                                                                                                                                                                                                                      * Also mark it as visited and crossed off list of unvisited nodes

Algorithms - RAM (Random Access Memory)
                                                                                                                                                                                                                                                                                                                                                         * Also known as working memory
                                                                                                                                                                                                                                                                                                                                                         * The place where variables are kept for use in functions 
                                                                                                                                                                                                                                                                                                                                                         * Numbered shelves where each shelf’s number is its address
                                                                                                                                                                                                                                                                                                                                                         * Each shelf holds 8 bits/1 byte  of binary numbers 
                                                                                                                                                                                                                                                                                                                                                         * Computer processor => memory controller => direct connection each address in RAM 
                                                                                                                                                                                                                                                                                                                                                         * Basically an array 


Algorithms - Cache
                                                                                                                                                                                                                                                                                                                                                         * Copies of things recently read from Random Access Memory
                                                                                                                                                                                                                                                                                                                                                         * Much faster to read than Random Access Memory 
                                                                                                                                                                                                                                                                                                                                                         * When processor asks for specific memory address, copies of nearby ones are also sent over for caching 


Algorithms - Integer Overflow 
                                                                                                                                                                                                                                                                                                                                                         * When you have the number 255 in an 8 bit unsigned integer in binary, and you add 1. The answer, 256 needs a 9th bit which can return an error or the computer can solve it, and toss the 9th bit, to give you zero instead. 


Algorithms - Fixed Width Integers: 
                                                                                                                                                                                                                                                                                                                                                         * Only 256 or 2^8 numbers can be expressed in one byte, so 4/2^32 or 8/2^64 bytes are often used for storing integers. 
                                                                                                                                                                                                                                                                                                                                                         * Most integers are fixed width/length and will take up the same amount of space in the RAM
                                                                                                                                                                                                                                                                                                                                                         * Fixed width integers take up O(1) constant space 
                                                                                                                                                                                                                                                                                                                                                         * Most simple operations on fixed width integers take O(1) time 
                                                                                                                                                                                                                                                                                                                                                         * Time and space efficient 


Algorithms - Pointers
                                                                                                                                                                                                                                                                                                                                                         * Say you want to store different things in an array but they are of different lengths
                                                                                                                                                                                                                                                                                                                                                         * Won’t be constant access time or space proficient


                                                                                                                                                                                                                                                                                                                                                         * It makes sense to just store the address in memory 
                                                                                                                                                                                                                                                                                                                                                         * This is just an array of integers which is accessible in O(1)
                                                                                                                                                                                                                                                                                                                                                         * Just points to another spot in memory 
                                                                                                                                                                                                                                                                                                                                                         * Not cache friendly 
                                                                                                                                                                                                                                                                                                                                                         * Slowdown not reflected in big O time cost 


Algorithms - Dynamic Arrays 
                                                                                                                                                                                                                                                                                                                                                         * For object oriented languages, arrays have to be created  with a specified number of indices 
                                                                                                                                                                                                                                                                                                                                                         * Dynamic Arrays - allow for resizing on demand 
                                                                                                                                                                                                                                                                                                                                                         * When resizing happens, a new bigger array twice as big is created 
                                                                                                                                                                                                                                                                                                                                                         * Each element from the old array is copied into the new array 
                                                                                                                                                                                                                                                                                                                                                         * Old array is freed up 
                                                                                                                                                                                                                                                                                                                                                         * New array can continue with appendings 
                                                                                                                                                                                                                                                                                                                                                         * A single doubling append is O(n) since you have to copy all n items from the array 
                                                                                                                                                                                                                                                                                                                                                         * An append on a dynamic array is always worst-case O(n) time 
                                                                                                                                                                                                                                                                                                                                                         * Through this each append has an average cost of O(1) 
                                                                                                                                                                                                                                                                                                                                                         * Some appends are just more expensive than others 




Logarithms - Solving for x when x is an exponent 




                                                                                                                                                                                                                                                                                                                                                         *                                                                                                                                                                                                                                                                                                                                                            * Logs are important in binary search because the total time cost of a binary search is O(log base 2 of n)
                                                                                                                                                                                                                                                                                                                                                         * Sorting cost O(n*log base 2 of n) in general

Section 27: UML (Unified Modeling Language)
Graphical language for constructing diagrams of a software system
                                                                                                                                                                                                                                                                                                                                                                                                                                                      * Using a set of diagrams such as class, activity, state, and sequence diagrams
                                                                                                                                                                                                                                                                                                                                                                                                                                                      * With these diagrams you can describe the structure, interaction, and behavior of different software components


Class diagrams help explain the relationships between classes and their responsibilities 
                                                                                                                                                                                                                                                                                                                                                                                                                                                      * Used to model the structure of a system 
                                                                                                                                                                                                                                                                                                                                                                                                                                                      * Also a useful source of documentation 
                                                                                                                                                                                                                                                                                                                                                                                                                                                      * Can also help analyze the flaws and weaknesses in a system 
                                                                                                                                                                                                                                                                                                                                                                                                                                                      *   =
Sequence diagrams - a type of interaction diagram that allows modeling of the flow of control and data as a sequence of messages flowing between objects 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Used to demonstrate the dynamic behavior of systems 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Help improve design of software 


Generalization and specialization
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Related to polymorphism and inheritance 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Generalization - starts with creating a parent/super class with all common properties that the related classes would share 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Specialization -  creating unique, specialized child classes with unique properties 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Derived from general class 
System entity - component of a system that represents an object or concept in the real world 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Ex: products, orders, users, etc
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Modeled as classes or objects 
Service class -  class that allows devs to set related operations as a service to  other systems or classes 
Different kinds of diagrams in UML
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Class diagram - shows the static structure of a system by modeling its classes, attributes, methods, and their relationships 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Attributes are classified as protected #, private -, or public + 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Use case diagram - describes the functionality of a system of by modeling actors, use cases and their relationships 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Has three parts: actor, scenario, and use case 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Primary actor - active participants in use case 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Secondary actor - passive participants in use case 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Sequence diagrams - shows interaction between objects in a system by modeling the messages exchanged between them 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Activity diagram - shows the workflow of a system by modeling activities, actions, and control flows 
Steps to officially create a UML
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Identify classes, attributes, and methods 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Create the UML class diagram 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Add classes, specify attributes and methods 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Add relationships between classes, such as associations, aggregations, and generalizations 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Refine the diagram by adding extra information such as constraints and stereotypes


How inheritance is represented - with a solid line, with an arrow pointing from the subclass to the superclass 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Arrow usually labeled with extends 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Sub class inherits all attributes, operations, and associations of the super class + it can have it’s own unique ones 




How state transition is represented - with a transition arrow that connects the source state to the target state 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Arrow can be labeled with an event that triggers the transition
How encapsulation is represented - by making the methods and attributes of a class private (-) and exposing only necessary public methods through which the object can be interacted with 
How to represent polymorphism - represented with with an arrow with open head pointing from the subclass to the parent class and the “extends” keyword 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * through inheritance and method overriding 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Inheritance - subclass inherits all methods and properties of super/parent class 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Overriding - subclass implements methods from its parent in its own unique way


How to represent concurrency - using a solid vertical line known as a lifeline and multiple dashed arrows extending from it 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Types of concurrency:
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Synchronous - multiple messages are executed in a synchronized way
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Asynchronous - multiple messages are executed in a non-synch way 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Combined fragments - represent complex interactions between lifelines and can be used to represent different types of concurrency like alternative and parallel behavior 


Modeling a database and different database diagrams        
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Entity Relationship Diagram - represents relationships between entities and attributes 
                                                                                                                                                                                                                                                                                                                                                                                                                                                         * Object role modeling diagram - represents objects, roles, and the relationships between them.


How to represent exception handling - handled using the “exception” keyword  followed by the name of the exception class

How to represent composition - diamond symbol with closed diamond
                                                                                                                                                                                                                                                                                                                                                                                                                                                            * “Part of relationship” - composed whole can’t exist without it’s parts and the same for the parts 
                                                                                                                                                                                                                                                                                                                                                                                                                                                            * A visitor center can have a bathroom and a lobby, but if the visitor center is closed or destroyed, both bathrooms and lobbies will be too


How to represent aggregation - diamond symbol with empty diamond

                                                                                                                                                                                                                                                                                                                                                                                                                                                            * A tortoise has a creep, but the creep can exist independently of the tortoise and the tortoise can do the same

Great Open Ended Interview Questions and Answers 


1. How do you ensure that your website design or web application is accessible and user-friendly?


I ensure that website designs and applications are accessible and user friendly by building my sites out with semantic HTML when I can, placing interactive elements higher up in the page, using headings and page titles and, labeling elements so that screen readers and other assistive technologies have an easier time navigating the page. I also run my sites through accessibility audit software for feedback and improvement suggestions.


2. What are your favorite features of HTML5 and CSS3, and how have you implemented them in your frontend development projects?


My favorite feature of HTML is semantic elements. Being able to use tags that reflect the content of a page make them so much easier to organize and design for. They also improve the SEO and accessibility so more potential users can be reached.


My favorite feature of CSS3 is the new nth child selector. It allows me to add more variability to page designs by being able to write more general CSS without having to dig deep down through parent and child selectors to get access to the elements I want.


3. What is Responsive Web Design?


Responsive Web Design is the school of thought that websites and apps should respond to user behavior and environment, including adjusting for screen size, and screen orientation. It's best demonstrated through the use of media queries, flexible grid layouts, and more.


4. What are your favorite types of frontend development projects to work on? What do you like about them?


My favorite types of projects to work on are social responsibility and social justice sites. I appreciate when my work brings awareness of different issues in the world and helps people to take action. I also enjoy projects that involve building libraries or documentation hubs to help people learn.


5. Why would you use a Javascript Library like ReactJs or Angular?


While a lot of base functionality for a site can be written in plain javascript or jQuery with CSS, predefined frameworks and libraries like React and Angular provide classes, methods, and helper functions to decrease development time. Frameworks also decrease DOM updates by using a virtual DOM to compare objects before updating. Frameworks also decrease development time through in-built features like data binding, routing and state management.
